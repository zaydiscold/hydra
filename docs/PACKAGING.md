# Packaging Guide

> **Single source of truth** for building, bundling, and distributing the Hydra Electron desktop app.

## Quick Start

```bash
# Full build chain (production frontend + resources + electron-builder)
npm run electron:build

# Smoke-test the unpacked artifact
npm run electron:smoke

# Or step-by-step:
npm run build                     # 1. Build Vite production frontend → dist/
npm run electron:prepare          # 2. Build empty DB + bundle Chromium
npx electron-builder              # 3. Package macOS/Windows/Linux artifact
```

Build output goes to `release/`.

---

## Scripts Overview

### `npm run electron:prepare`

Entry point: `scripts/prepare-electron-resources.mjs`

Prepares two assets consumed by `electron-builder` via `extraResources`:

| Output | Source | Purpose |
| --- | --- | --- |
| `build/electron/data/empty-hydra.db` | Generated by `scripts/build-empty-db.mjs` | Pre-initialized SQLite with all tables (zero rows) |
| `build/electron/chromium/` | Playwright cache (`~/.cache/ms-playwright` or `~/Library/Caches/ms-playwright`) | Platform-specific Chromium binary |

**Chromium resolution order:**
1. `PLAYWRIGHT_BROWSERS_PATH` environment variable
2. Platform-native cache: `~/Library/Caches/ms-playwright` (macOS) / `~/.cache/ms-playwright` (Linux) / `%LOCALAPPDATA%/ms-playwright` (Windows)
3. If not found, runs `npx playwright install chromium` to download it

Only the current platform's Chromium variant is copied (e.g. `chrome-mac-arm64` on Apple Silicon, `chrome-linux` on Linux, `chrome-win` on Windows).

### `npm run electron:build`

Full chain:
1. `npm run build` — Vite production build → `dist/`
2. `npm run electron:prepare` — Resources for `extraResources`
3. `electron-builder` — Package using `electron-builder.yml`

### `npm run electron:smoke`

Entry point: `scripts/smoke-electron-package.mjs`

Verifies the unpacked package contract without launching a GUI. Checks:
- `resources/prisma/schema.prisma` exists
- `resources/prisma/migrations/` exists
- `resources/data/empty-hydra.db` exists
- `.prisma/client/` with a native query engine binary exists in `resources/app/node_modules/`
- `resources/chromium/` contains a known Chromium executable path
- App size ≤ `HYDRA_MAX_PACKAGED_APP_MB` (default 900 MB)
- `PrismaClient.$queryRaw\`SELECT 1\`` succeeds against the empty DB

Usage:
```bash
node scripts/smoke-electron-package.mjs
# Or with a specific resources path:
ELECTRON_APP_RESOURCES=/path/to/Hydra.app/Contents/Resources node scripts/smoke-electron-package.mjs
```

## Packaging Architecture

### electron-builder.yml

Key configuration (`electron-builder.yml`):

```
asar: false          — Prisma's .prisma/client lookup requires un-archived node_modules
afterPack: ./electron/builders/afterPack.js   — .prisma copy + engine pruning
```

**extraResources** (copied into the app bundle's `Resources/` directory):

| Source | Destination (in Resources/) |
| --- | --- |
| `build/electron/data/` | `data/` |
| `build/electron/chromium/` | `chromium/` |
| `prisma/schema.prisma` | `prisma/schema.prisma` |
| `prisma/migrations/` | `prisma/migrations/` |

**files** (what ships inside the app bundle):

```
dist/**/*                          — Vite-built frontend
server/**/*                        — Express backend (excl. tests/scripts)
prisma/schema.prisma               — Schema for self-heal
prisma/migrations/**/*             — Migration SQL for self-heal
electron/**/*                      — Main process + preload (excl. tests)
package.json                       — Entry point + version
**/node_modules/.prisma/**/*       — Prisma generated client
**/node_modules/@prisma/**/*       — Prisma runtime
```

### afterPack Hook

File: `electron/builders/afterPack.js`

Runs after electron-builder's default pruning pass. Does two things:

1. **Copies `.prisma/client/`** from the repo's `node_modules/.prisma/` into the packaged `Resources/app/node_modules/`, filtering to keep only the native query engine for the **target platform** (e.g. `libquery_engine-darwin-arm64.dylib.node` on macOS ARM64). Foreign platform engines (~20 MB each) are dropped.

2. **Prunes unused Prisma runtime variants** — removes wasm, edge-runtime, browser, react-native, and source map files from `@prisma/client/runtime/`.

### Platform Targets

| Platform | Format | Arch | Artifact Name |
| --- | --- | --- | --- |
| macOS | DMG | arm64 | `Hydra-{version}-arm64.dmg` |
| Windows | NSIS installer | x64 | `Hydra-Setup-{version}-x64.exe` |
| Linux | AppImage | x64 | `Hydra-{version}-x64.AppImage` |
| Linux | deb | x64 | `hydra_{version}_amd64.deb` |

Output directory: `release/`

## Runtime Resource Flow

When the packaged app launches, `ensurePackagedRuntimeState()` in `electron/main.js` runs:

```
┌─────────────────────────────────────────────────┐
│ 1. Pin ENV:                                    │
│    HYDRA_DATA_DIR = app.getPath('userData')     │
│    DATABASE_URL = file:userData/hydra.db        │
│    HYDRA_EMBEDDED = 1                           │
└──────────────┬──────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────────┐
│ 2. Per-install JWT secret                       │
│    userData/jwt-secret                          │
│    (32-byte random hex, auto-generated)         │
│    → process.env.JWT_SECRET                     │
└──────────────┬──────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────────┐
│ 3. Empty DB copy (first launch only)            │
│    resourcesPath/data/empty-hydra.db            │
│    → userData/hydra.db                          │
└──────────────┬──────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────────┐
│ 4. Schema self-heal (firstLaunchSetup)          │
│    Compares schema content hash against         │
│    userData/.schema-version sentinel            │
│    If different:                                │
│      a. Try prisma db push (dev only)           │
│      b. Try npx prisma (dev only)               │
│      c. self-heal: replay ALTER TABLE +         │
│         CREATE INDEX idempotently via           │
│         server/lib/db-self-heal.js              │
│    Write new hash → .schema-version             │
└──────────────┬──────────────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────────────┐
│ 5. Chromium resolution (on demand)              │
│    playwright-browser.js scans:                 │
│    resourcesPath/chromium/Chromium.app/...      │
│    resourcesPath/chromium/chrome-mac/...        │
│    resourcesPath/chromium/chrome-linux/chrome   │
│    resourcesPath/chromium/chrome-win/chrome.exe │
└─────────────────────────────────────────────────┘
```

## Empty Database

### How It's Built

`scripts/build-empty-db.mjs` generates `data/empty-hydra.db`:

1. **Primary method:** Runs `npx prisma db push --accept-data-loss --force-reset` against a temporary SQLite file with `DATABASE_URL=file:<temp>`.
2. **Fallback method:** If Prisma CLI is unavailable, runs `sqlite3` directly with a hardcoded bootstrap SQL that mirrors all Prisma schema tables, indexes, and foreign keys.
3. The temp DB is copied to `data/empty-hydra.db` and cleaned up.

### How It's Used

- **Script phase:** `build-empty-db.mjs` outputs to `data/empty-hydra.db`
- **Prepare phase:** `prepare-electron-resources.mjs` copies it to `build/electron/data/empty-hydra.db`
- **Build phase:** `electron-builder.yml` picks it up via `extraResources` → `Resources/data/empty-hydra.db`
- **Runtime:** `electron/main.js` copies it to `userData/hydra.db` if the file doesn't exist yet

## Chromium Bundling

### How It's Bundled

`scripts/prepare-electron-resources.mjs`:

1. Reads `playwright-core/browsers.json` to find the pinned Chromium revision.
2. Searches Playwright's cache directories for the `chromium-{revision}` folder.
3. Copies only the current platform's subdirectory (`chrome-mac-arm64` on Apple Silicon, `chrome-mac` on Intel macOS, `chrome-linux` on Linux, `chrome-win` on Windows).
4. Validates that at least one known executable path exists in the output.
5. electron-builder copies `build/electron/chromium/` to `Resources/chromium/` via `extraResources`.

### How It's Resolved at Runtime

`server/lib/playwright-browser.js` → `resolveBundledChromium()`:

Scans `process.resourcesPath/chromium/` for known executable layouts in priority order:

| Platform | Candidate paths |
| --- | --- |
| macOS | `chromium/Chromium.app/Contents/MacOS/Chromium` |
| macOS | `chromium/chrome-mac/Chromium.app/Contents/MacOS/Chromium` |
| macOS | `chromium/chrome-mac-arm64/Chromium.app/Contents/MacOS/Chromium` |
| macOS | `chromium/chrome-mac/Google Chrome for Testing.app/...` |
| macOS | `chromium/chrome-mac-arm64/Google Chrome for Testing.app/...` |
| Linux | `chromium/chrome-linux/chrome` |
| Windows | `chromium/chrome-win/chrome.exe` |
| Generic | `chromium/chrome`, `chromium/chromium`, `chromium/chromium-browser` |

If bundled Chromium is not found, falls back to `channel: chrome` (system Google Chrome).

### Environment Overrides

| Variable | Effect |
| --- | --- |
| `HYDRA_PLAYWRIGHT_EXECUTABLE_PATH` | Explicit binary path (highest priority) |
| `HYDRA_PLAYWRIGHT_CHANNEL` | System browser channel (e.g. `chrome`) |
| `HYDRA_PLAYWRIGHT_CDP_ENDPOINT` | Remote debugging endpoint (bypasses launch entirely) |
| `HYDRA_PLAYWRIGHT_HEADED` | Set to `true` to show the browser window |

## Prisma in Packaged App

Three mechanisms ensure Prisma works in the packaged app:

### 1. asar: false
The asar archive is disabled because `@prisma/client/default.js` does `require('.prisma/client')` which walks up `node_modules` looking for a dotfile directory. Electron-builder's default glob filtering excludes dotfiles, and there's no reliable YAML pattern to opt them back in.

### 2. Explicit file includes
`electron-builder.yml` lists:
```yaml
files:
  - "**/node_modules/.prisma/**/*"
  - "**/node_modules/@prisma/**/*"
asarUnpack:
  - "**/node_modules/.prisma/**/*"
  - "**/node_modules/@prisma/**/*"
```
This ensures the generated client and native engine survive packaging.

### 3. afterPack hook
`electron/builders/afterPack.js` copies `.prisma/client/` into the packaged `node_modules` and prunes:
- Foreign-platform engine binaries (saves ~60-80 MB)
- WASM and edge-runtime variants
- Browser and React Native shims
- Source map files

## Code Signing

### macOS

Configuration (`electron-builder.yml`):
```yaml
mac:
  hardenedRuntime: true
  gatekeeperAssess: false
  entitlements: desktop/entitlements.mac.plist
  entitlementsInherit: desktop/entitlements.mac.plist
```

To sign, set environment variables before building:
```bash
export CSC_LINK=/path/to/DeveloperIDApplication.p12
export CSC_KEY_PASSWORD=your-password
npm run electron:build
```

Without signing, macOS users must **Right-click → Open** on first launch.

### Windows

`electron-builder.yml` has `verifyUpdateCodeSignature: false`. To sign:
```bash
export CSC_LINK=/path/to/authenticode.p12
export CSC_KEY_PASSWORD=your-password
```

### Linux

Linux packages (AppImage, deb) do not require signing.

## Smoke Tests

### What Gets Verified

`scripts/smoke-electron-package.mjs` checks:

| Check | What it asserts |
| --- | --- |
| Resources directory | Found under `release/` for the current platform |
| Prisma schema | `resources/prisma/schema.prisma` exists |
| Prisma migrations | `resources/prisma/migrations/` exists and is a directory |
| Empty DB | `resources/data/empty-hydra.db` exists |
| Prisma engine | `.prisma/client/` has a `query_engine-*.node` binary |
| Bundled Chromium | At least one known Chromium executable under `resources/chromium/` |
| App size | ≤ 900 MB (configurable via `HYDRA_MAX_PACKAGED_APP_MB`) |
| DB connectivity | `PrismaClient.$queryRaw\`SELECT 1\`` succeeds against the empty DB |

### Running

```bash
# After npm run electron:build
npm run electron:smoke

# With a custom resources path
ELECTRON_APP_RESOURCES=/path/to/Hydra.app/Contents/Resources node scripts/smoke-electron-package.mjs
```

```
│       │   ├── electron/            # Main process + preload
│       │   ├── prisma/

│       │   │   └── migrations/      # (also in extraResources)

│       │   │   ├── .prisma/client/  # Generated client + native engine
│       │   │   └── @prisma/client/  # Prisma runtime (pruned)
│       │   └── package.json
│       ├── data/
│       │   └── empty-hydra.db       # Pre-initialized blank database

│       │   └── chrome-mac-arm64/    # Platform-specific Playwright Chromium

│           ├── schema.prisma        # Schema for self-heal
│           └── migrations/          # Migration SQL for self-heal
```
